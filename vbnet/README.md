# Auto Generate Diameter Sheet

ระบบสร้างรายงาน Diameter อัตโนมัติ พัฒนาด้วย VB.NET, .NET 8 และ Windows Forms

## 1. Overviews

โปรเจคนี้พัฒนาขึ้นเพื่อเปลี่ยนกระบวนการจัดทำรายงาน Diameter จากการเขียนและจัดข้อมูลด้วยมือ ให้เป็นโปรแกรมที่สามารถโหลดข้อมูล จัดวางข้อมูลลงรายงาน และตรวจสอบกับใบ Manual ได้

ความสามารถหลัก:

- โหลดข้อมูลจาก MySQL ด้วย Production Place Code และ Coat Lot Number
- Import ข้อมูลจากไฟล์ XLSX
- Import ข้อมูลจากรูปภาพ Manual แล้วตรวจแก้ก่อนสร้างรายงาน
- จัดข้อมูลลงช่อง `T-1` ถึง `T-22` ตาม Tray Lot และ Logic ของกระบวนการ
- รองรับรายงานมากกว่า 1 หน้าเมื่อข้อมูลเกิน 22 ตำแหน่ง
- แก้ไข Header เช่น ITEM, DOME Type, CAPA และ DATETIME
- Zoom ด้วย Scroll Mouse และเลื่อนหน้ารายงานด้วยการกดเมาส์ซ้ายค้าง
- Compare ข้อมูลในตารางโปรแกรมกับรูปใบ Manual ตามตำแหน่ง `T / Row / R-L`
- Resume ผล Compare เดิมภายใน session โดยไม่ต้อง OCR รูปใหม่
- สร้าง Installer สำหรับแจกภายในบริษัทด้วย Inno Setup

Technology ที่ใช้:

- VB.NET และ .NET 8
- Windows Forms
- MySqlConnector
- ClosedXML
- Tesseract OCR
- MSTest
- Inno Setup 6

ข้อจำกัดของ OCR:

- OCR ใช้ช่วยตรวจสอบ ไม่ใช้เป็นผลยืนยันสุดท้าย
- ลายมือจาง ภาพเบลอ แสงสะท้อน และเส้นตาราง อาจทำให้ OCR อ่านผิดหรืออ่านไม่พบ
- Compare ไม่แก้ข้อมูลในรายงานหลักหรือ Database

Tesseract OCR ถูก bundle มากับโปรแกรมและ Installer ผู้ใช้ไม่ต้องติดตั้ง Tesseract แยก โดยโปรแกรมจะค้นหาตามลำดับ:

1. `tesseract/tesseract.exe` ภายในโฟลเดอร์โปรแกรม
2. `C:\Program Files\Tesseract-OCR\tesseract.exe`
3. `C:\Program Files (x86)\Tesseract-OCR\tesseract.exe`

### `AutoGenerateDiameter.sln`

Solution หลักสำหรับเปิดโปรเจคทั้งหมดใน Visual Studio ประกอบด้วย Core, WinForms และ Tests

### `AutoGenerateDiameter.Core`

เก็บ Models และ Business Logic ที่ไม่ผูกกับหน้าจอ

| ไฟล์ | หน้าที่ |
|---|---|
| `Models.vb` | โครงสร้างข้อมูล เช่น Measurement, SheetData, SourceRecord และการตั้งค่า MySQL |
| `MySqlReportLoader.vb` | เชื่อมต่อ MySQL รัน SQL และโหลดข้อมูลสำหรับสร้างรายงาน |
| `XlsxReportImporter.vb` | อ่านและ Map ข้อมูลจากไฟล์ XLSX |
| `ReportBuilder.vb` | จัดกลุ่ม Tray และสร้าง Sheet ตาม Logic ของกระบวนการ |
| `ManualImageImportService.vb` | อ่านข้อมูลจากรูป Manual และสร้างข้อมูลสำหรับ Import |
| `ManualCompareService.vb` | OCR รูปหลายรอบและ Compare กับข้อมูลโปรแกรมตามตำแหน่ง |
| `SampleReportFactory.vb` | สร้างข้อมูลตัวอย่างสำหรับการพัฒนาหรือทดสอบ |

### `AutoGenerateDiameter.WinForms`

เก็บ Desktop UI และการเชื่อมการทำงานจากผู้ใช้ไปยัง Core

| ไฟล์ | หน้าที่ |
|---|---|
| `Program.vb` | จุดเริ่มต้นของโปรแกรม |
| `Form1.vb` | หน้าหลัก ปุ่มคำสั่ง การโหลดข้อมูล และ session ของ Manual Compare |
| `Form1.Designer.vb` | ส่วนประกอบพื้นฐานของหน้าหลัก |
| `Dialogs.vb` | Dialog สำหรับ MySQL, Import Manual, Edit Header, Compare และเลือก Format |
| `ReportPreviewControl.vb` | วาดรายงาน ตาราง Header/Footer รองรับ Zoom และ Pan |

ปุ่มหลักบนหน้าจอ:

| ปุ่ม | หน้าที่ |
|---|---|
| `Load Database` | โหลดข้อมูลจาก MySQL |
| `Import XLSX` | โหลดข้อมูลจากไฟล์ XLSX |
| `Import Manual Image` | อ่านรูป Manual และสร้างรายงานจากข้อมูลที่ตรวจแก้แล้ว |
| `Edit Header` | แก้ ITEM, DOME Type, CAPA และ DATETIME |
| `Compare Manual` | OCR รูปและเปรียบเทียบกับตารางปัจจุบัน |
| `Resume Compare` | กลับไปตรวจผล Compare เดิมใน session |

### `AutoGenerateDiameter.Tests`

เก็บ Unit Tests ของ Logic หลัก

| ไฟล์ | สิ่งที่ทดสอบ |
|---|---|
| `ReportBuilderTests.vb` | Logic จัดกลุ่ม Tray และสร้าง Report |
| `XlsxReportImporterTests.vb` | การ Import และ Map XLSX |

### `installer` และ `build-installer.ps1`

- `installer/AutoGenerateDiameter.iss` คือ Script ของ Inno Setup
- `installer/output/AutoGenerateDiameterSetup-x64.exe` คือ Installer ที่สร้างเสร็จ
- `build-installer.ps1` ใช้ Publish โปรแกรมและสร้าง Installer อัตโนมัติ

### `third_party/tesseract`

เก็บ Tesseract Portable runtime, DLL dependencies, `eng.traineddata` และ License โปรแกรมจะคัดลอกโฟลเดอร์นี้ไปยัง Build, Publish และ Installer อัตโนมัติ

### โฟลเดอร์ที่สร้างอัตโนมัติ

- `bin/` และ `obj/`: ไฟล์จากการ Build
- `publish/`: ไฟล์โปรแกรมสำหรับแจก
- `verify_output/`: output ชั่วคราวสำหรับตรวจ Compile

## 3. Flow การทำงาน

### Flow หลักของโปรแกรม

```text
เปิดโปรแกรม
    ↓
เลือกแหล่งข้อมูล
    ├─ Load Database
    ├─ Import XLSX
    └─ Import Manual Image
    ↓
Core แปลงข้อมูลเป็น SourceRecord / Measurement
    ↓
ReportBuilder จัดกลุ่มข้อมูลตาม Tray Logic
    ↓
สร้าง SheetData หน้า 1..N
    ↓
ReportPreviewControl แสดงรายงาน
    ↓
ผู้ใช้ Zoom, Pan, เปลี่ยนหน้า หรือ Edit Header
```

### Flow: Load Database

```text
กด Load Database
    ↓
กรอก Host, Port, Database, Username, Password
    ↓
กรอก PPC และ CLN
    ↓
MySqlReportLoader รัน SQL
    ↓
ReportBuilder จัดข้อมูลลง T-1..T-22
    ↓
แสดงรายงาน
```

### Flow: Import XLSX

```text
กด Import XLSX
    ↓
เลือกไฟล์ .xlsx
    ↓
XlsxReportImporter อ่านชื่อ Column และข้อมูล
    ↓
Map ข้อมูลเป็น SourceRecord
    ↓
ReportBuilder สร้างรายงานใหม่แทนข้อมูลเดิม
```

### Flow: Import Manual Image

```text
กด Import Manual Image
    ↓
เลือกรูปใบ Manual
    ↓
Tesseract OCR อ่านตัวเลข
    ↓
Map ตำแหน่งเป็น T / Row / R-L
    ↓
เปิดตารางให้ผู้ใช้ตรวจแก้
    ↓
กด Import Reviewed Data
    ↓
สร้างรายงานใหม่จากข้อมูลในรูป
```

### Flow: Compare Manual

Compare ใช้ตรวจสอบข้อมูลเท่านั้น ไม่แก้รายงานหลัก

```text
มีข้อมูลแสดงในรายงานปัจจุบัน
    ↓
กด Compare Manual และเลือกรูป
    ↓
OCR ตัวเลขหลายรอบ
    ├─ Numeric: PSM 3, 6, 11, 12
    └─ Text: PSM 6, 11
    ↓
รวมผล OCR ที่อ่านเหมือนกันและตำแหน่งใกล้กัน
    ↓
Compare กับ Expected ตามตำแหน่ง T / Row / R-L
    ↓
แสดงภาพเต็ม กรอบตำแหน่ง และภาพขยาย
    ↓
ผู้ใช้เลือกหนึ่งหรือหลายรายการ
    ↓
กด Confirm selected rows
    ↓
สถานะเปลี่ยนเป็น MANUALLY CONFIRMED
```

สถานะ Compare:

| สถานะ | ความหมาย |
|---|---|
| `MATCH` | OCR อ่านได้และค่าตรงกับโปรแกรม |
| `MISMATCH` | OCR อ่านได้แต่ค่าไม่ตรง |
| `MISSING` | โปรแกรมมีค่า แต่ OCR อ่านจากรูปไม่พบ |
| `LOW CONFIDENCE` | OCR อ่านได้แต่ confidence ต่ำกว่า 40% |
| `UNEXPECTED VALUE` | OCR พบค่าในช่องที่โปรแกรมว่าง |
| `NOT AVAILABLE` | Format ของรูปไม่มีข้อมูลประเภทนั้น |
| `MANUALLY CONFIRMED` | ผู้ใช้ตรวจภาพและยืนยันรายการแล้ว |

ถ้าปิดหน้าต่าง Compare:

- ผล Confirm และ Pending ยังอยู่ใน session
- กด `Resume Compare` เพื่อกลับไปตรวจต่อโดยไม่ OCR ใหม่
- โหลดข้อมูลใหม่ เปลี่ยนหน้า หรือปิดโปรแกรม จะล้างผล Compare เดิม

### Flow: สร้าง Installer

ต้องติดตั้ง Inno Setup 6 ที่ `D:\Inno Setup 6`

```powershell
cd D:\hoya\Auto_generate_diameter_sheet\vbnet
.\build-installer.ps1 -Version "1.1.0"
```

Script จะทำงานตามลำดับ:

```text
dotnet publish แบบ Release และ self-contained
    ↓
สร้างไฟล์ใน publish/win-x64
    ↓
Inno Setup compile Installer
    ↓
installer/output/AutoGenerateDiameterSetup-x64.exe
```
